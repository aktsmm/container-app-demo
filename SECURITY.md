# セキュリティポリシー

本リポジトリは Azure 上のデモ環境向けハンズオン資料です。セキュリティ報告は GitHub Issue のみで受け付け、メールなどの個別対応は行いません。デモ用途につき、修正対応もベストエフォートとなります。

## 対象範囲

- Azure Kubernetes Service (AKS) 上で稼働する掲示板アプリケーション
- Azure Container Apps 上で稼働する管理アプリケーション
- Azure Virtual Machine 上で動作する MySQL、およびそのバックアップ/ストレージ構成
- GitHub Actions ワークフロー、Bicep テンプレート、サンプルアプリケーションコード

## 報告フロー

1. GitHub の **Issue**（「Security」ラベルを追加）にて報告してください。
2. Issue に投稿できない機微情報は共有しないでください（別経路の受付はありません）。
3. 返信 SLA は定めておらず、対応はベストエフォートです。

## サポートポリシー

- 本リポジトリはメンテナの学習用サンプルのため、公式サポートは提供しません。
- Issue への回答や修正は時間がある範囲で行いますが、対応を保証できません。

## 協力いただきたい点

- 未認可アクセス試行を行う際は、Azure 利用規約や各種法律を遵守してください。
- DDoS、データ破壊、大量トラフィック生成など本番環境に悪影響を及ぼす手法は使用しないでください。
- 攻撃コードを第三者へ共有する前に、メンテナと十分に調整してください。
